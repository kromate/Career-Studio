# Security Policy

## Reporting a Vulnerability

Do not report vulnerabilities or exposed credentials in a public issue.

Before the repository is published, replace this section with a monitored
private security email or GitHub private vulnerability reporting instructions.

## Sensitive Data

Resumes, job descriptions, generated documents, account IDs, and analysis
results may contain personal or confidential information.

Contributors must not:

- commit real user resumes or production exports;
- log resume text, tokens, or credentials;
- include personal data in analytics events or test snapshots;
- use production data as public fixtures;
- send user documents to an undeclared AI provider.

Use synthetic or explicitly consented, anonymized fixtures for testing.
